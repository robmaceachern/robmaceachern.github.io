A few weeks ago, Marco Arment posted about launching an accidental iPad version of his non-universal app, Overcast. I ran into the same problem while setting up a storyboard launch image for an iPhone-only app and I've discovered a simple workaround for the problem.

In your Info.plist file, append ~iphone to the UILaunchStoryboardName key. Done!

Now when you run your iPhone only app on an iPad, it will load in the classic iPhone windowed mode instead treating it like a universal app.

 

I also experimented with setting different launch screen storyboards for the iPad by creating a UILaunchStoryboardName~ipad key in the plist and it seemed to work fine too, which may come in handy if you want to use different launch storyboards on iPhone and iPad when making universal apps.

I've created a basic iOS project on Github that you can demo. Give it a shot.
